Github+Gridea创建个人Blog

· 2021-11-06 · # 教程

最初采用 Github+Hexo 的方式创建了Blog，后来发现从配置到最后写文章发布都比较繁琐，偶然的机会了解到 Gridea（一个静态博客写作客户端，带有可视化界面），可以大大简化的建立blog的流程。这里主要总结分享一下采用 Github+Gridea的方式建立个人Blog的简易方式，主要步骤：

  • 注册Github
  • 购买域名
  • 配置Gridea
  • 更换主题

Github

  1. 注册GitHub。官网注册
  2. 创建仓库｜Create a repository。用于存放博客数据，这里需要注意一点，仓库名称需要固定的格式“username.github.io”，这里的username要和第1步注册时用户名保持一致（如果不购买域名，这个地址也就是博客的登录的地址，介意的话用户名一定要提前想好）
  3. 生成Token。Github登录账号后，右上角头像，依次点击 Settings-Developer setting-Personal access tokens-Create new token（这里日期可以设置为无限制）。作用主要是后面配置Gridea使用，可以理解为本地文章向Github提交数据时的认证

购买域名

  1. 阿里云购买。其他方式也可以，价格也不贵，注册自己喜欢的，这里不赘述了（另外，注册域名需要实名认证，大概1～3个工作日）
  2. 域名解析。主要就是将域名指定到GitHub地址，参考下图

配置Gridea

  1. 去官网下载Gridea客户端
  2. 配置。参考下图（要注意，目前GitHub默认创建仓库后到分支名称是main，而不是master），令牌就是Token，配置完成后远程检测测试一下，成功后保存

更换主题

  1. 依次打开 主题-更多主题，找到自己喜欢的主题后，下载到本地解压缩（参考下图）
  2. 在下图左下角位置找到本地存放文件数据位置，将解压缩后的文件移动到/*/Gridea/themes下，重启Gridea后就可以在主题中更换刚刚下载的主题

这样，博客就算搭建好了。
